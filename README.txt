=====================================
  AutoCamera Monitor
=====================================

Автоматическая проверка камер видеонаблюдения по расписанию
с отправкой отчётов на email.


ЧТО ДЕЛАЕТ
-----------
Скрипт проверяет все камеры по 8 системам видеонаблюдения:
  - Онлайн / Офлайн статус
  - Наличие записи
  - Свежесть файлов записей (SMB-шары)

По результатам:
  - Формирует HTML-отчёт и отправляет на email (по группам: Европласт, Онлайн)
  - Если есть поломки — отдельное письмо на helpdesk


СИСТЕМЫ КАМЕР
-------------
  Система                  Тип проверки       Камер
  -------                  ------------       -----
  Производство (TRASSIR)   HTTP API           ~24
  Офис (iPanda)            RTSP               16
  Склад (HiWatch)          ISAPI              16
  Новый цех (iPanda)       RTSP               10
  Стройка (записи)         SMB-шара           3 канала
  Цех выдува (HiWatch)     ISAPI              16
  BEWARD (удалённые)       SMB-шара           7
  iVMS (Hikvision)         ISAPI              3
  Ростелеком               Портал RT + Ping   7


БЫСТРЫЙ СТАРТ
--------------
  1. Установи зависимости:     npm install
  2. Заполни .env (учётные данные камер, SMTP, адреса email)
  3. Запусти:                  AutoCamera.bat


МЕНЮ (AutoCamera.bat)
---------------------
Интерактивное меню с пунктами:

  1      Полная проверка всех систем + отправка email
  2-10   Проверка одной системы (без email)
  R      Открыть последний отчёт в браузере
  L      Открыть папку логов
  S      Настроить расписание автозапуска
  E      Изменить email-адреса (отчёт / helpdesk)

В шапке меню отображаются текущие настройки:
адреса email и статус расписания.


ЗАПУСК ИЗ КОМАНДНОЙ СТРОКИ
---------------------------
  node src/index.js                            полная проверка + email
  node src/index.js --dry-run                  проверка без отправки email
  node src/index.js --dry-run --only trassir   только одна система
  node src/index.js --debug                    подробные логи
  node src/index.js --test-email               тестовый email


РАСПИСАНИЕ
----------
Настраивается через меню (пункт S) или вручную:
  - config/schedule.json   — время запуска (МСК), интервал повтора
  - setup-schedule.ps1     — создаёт задачу в Windows Task Scheduler

Пример: запуск каждый день в 10:00 МСК, повтор каждые 4 часа.


EMAIL-УВЕДОМЛЕНИЯ
-----------------
Отчёт:
  Полный HTML-отчёт по всем камерам, отправляется на REPORT_TO.
  Формируется по группам (Европласт / Онлайн) — два отдельных письма.

Helpdesk:
  Отправляется на HELPDESK_TO только если есть сломанные камеры.
  Содержит таблицу: система, камера, статус, подробности.
  Камеры из списка helpdeskIgnore в config/systems.json исключаются.

Адреса меняются через меню (пункт E) или прямо в .env.


СТРУКТУРА ФАЙЛОВ
----------------
  autocamera/
    AutoCamera.bat          запуск меню
    menu.ps1                интерактивное меню (PowerShell)
    setup-schedule.ps1      настройка планировщика Windows
    .env                    учётные данные и настройки
    config/
      systems.json          список камерных систем и параметры
      schedule.json         расписание автозапуска
    src/
      index.js              главный пайплайн проверки
      reporter.js           HTML-отчёт + email + helpdesk
      logger.js             логирование
      isapi.js              Hikvision/HiWatch ISAPI проверка
      rtsp-check.js         RTSP-проверка камер iPanda через NVR
      trassir-check.js      TRASSIR SDK HTTP API
      hikvision-multi.js    проверка одиночных Hikvision-камер (iVMS)
      beward-check.js       BEWARD — SMB freshness + ping
      recordings-check.js   проверка записей на SMB-шаре
      rostelecom-check.js   портал РТ (headless Playwright)
      smb-utils.js          SMB-хелперы (подключение, диагностика)
    logs/                   логи (хранятся 14 дней)
    reports/                HTML-отчёты (хранятся 14 дней)


ТРЕБОВАНИЯ
----------
  - Windows 10/11 x64
  - Node.js v18+
  - Доступ к камерам по сети (LAN / маршруты)
  - Playwright Chromium (для Ростелеком-портала)
