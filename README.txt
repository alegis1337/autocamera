=====================================
  AutoCamera Monitor v2.2
=====================================

Автоматическая проверка камер видеонаблюдения по расписанию
с отправкой отчётов на email и заявок в helpdesk.


ЧТО ДЕЛАЕТ
-----------
Проверяет все камеры по 9 системам видеонаблюдения:
  - Онлайн / Офлайн статус
  - Наличие записи
  - Свежесть и качество SMB-записей (Стройка / BEWARD)

По результатам:
  - Формирует HTML-отчёт со скриншотами каждой камеры
  - Рассылает по группам (Европласт, Онлайн)
  - На helpdesk шлёт письмо ТОЛЬКО при появлении НОВЫХ поломок
    (восстановления — только в почтовом отчёте, helpdesk не дёргают).
    В письме перечислены все актуально сломанные камеры группы.
  - Накапливает журнал событий для отчёта за период

РЕЖИМЫ ЗАПУСКА
---------------
Два режима через CLI-флаги:

  node src/index.js --light
       Лёгкий прогон каждые 15 мин:
       только статусы + timeline + live.html
       НЕ снимает кадры, НЕ шлёт email/helpdesk

  node src/index.js --daily
       Полный прогон раз в день (по умолчанию 14:00 МСК):
       статусы + снимки + email + helpdesk

  node src/index.js
       Ручной запуск (то же что --daily)

Дополнительные флаги:
  --dry-run           без отправки email
  --debug             подробные логи
  --only <id>         одна система (trassir, ipanda-office, и т.д.)
  --reset-state       обнулить helpdesk-state (все broken будут "новыми")
  --no-snapshots      пропустить снимки + Bitrix Диск
  --test-email        отправить пустое тестовое письмо


АВТОЗАПУСК (Windows Task Scheduler)
-----------------------------------
Две задачи создаются автоматически через setup-schedule.ps1
(или из меню — пункт S):

  AutoCamera Light  — каждые 15 мин с 06:00 по 20:00 МСК
  AutoCamera Daily  — раз в день в 14:00 МСК

Настройка в config/schedule.json:
  {
    "light": { "taskName": "AutoCamera Light", "intervalMinutes": 15,
               "startTimeMSK": "06:00", "durationHours": 14 },
    "daily": { "taskName": "AutoCamera Daily", "startTimeMSK": "14:00" }
  }


МЕНЮ (menu.ps1)
---------------

  Проверки:
    1      Полная проверка всех систем (--daily)
    T      Тестовая проверка ВСЕХ (dry-run)
    2-10   Проверка одной системы (Производство/Офис/Склад/Новый цех/
           Стройка/Цех выдува/BEWARD/iVMS/Ростелеком)

  Просмотр:
    R      Открыть последний отчёт в браузере
    V      Открыть live-monitor (auto-refresh 30 сек)
    H      Отчёт за период (день/3д/7д/30д/произвольный)
    L      Открыть папку логов

  Настройка:
    S      Настроить расписание (light + daily)
    E      Email-адреса (Европласт / Онлайн / Helpdesk)
    G      Управление камерами:
              - снять/поставить серый маркер
              - удалить камеру/канал
              - "A" — Мастер добавления нового устройства

    0      Выход


ДОБАВЛЕНИЕ НОВОГО УСТРОЙСТВА
-----------------------------
Мастер запускается из меню → G → A.

Спрашивает: IP, порт, логин, пароль.
Автоматически пробует:
  - Hikvision/HiWatch ISAPI (порт 80)
  - TRASSIR SDK (порт 8080)
  - BEWARD (порт 80)
  - RTSP (порт 554) как фолбэк

Определяет: тип устройства, модель, число каналов.
Просит подтверждения и записывает в systems.json + .env (логин/пароль).


ОТЧЁТ ЗА ПЕРИОД
---------------
Меню → H → выбрать период.

Скрипт читает все state/timeline-YYYY-MM-DD.json за диапазон,
агрегирует инциденты по каждой камере, формирует HTML с разбивкой:

  - Сводка: камер с проблемами, всего падений, суммарный простой
  - Группировка по объектам
  - По каждой камере: список периодов (с указанием времени) по дням
    Пример: "19.05: 14:20 — 15:50 (1ч 30м); 20.05: 11:40 — сейчас"


СКРИНШОТЫ В ОТЧЁТЕ
-------------------
Каждая камера в гриде отчёта показывается миниатюрой:

  - Online камера → свежий кадр или последний рабочий (last-good кэш)
  - Offline → последний рабочий снимок (чтобы понять что за камера)
  - Серая (unused) → пустой серый квадрат + подпись
  - SMB-системы (Стройка, BEWARD) → без миниатюр, как в v1

Полноразмерные снимки + ежедневная архивация в Bitrix24 Диск:
  /AutoCamera/<Группа>/<Объект>/<YYYY-MM-DD>/<имя_камеры>.jpg

Локальный кэш миниатюр:
  screenshots/last-good/<systemId>/<index>-<имя>.jpg

ТРЕБОВАНИЯ
----------
  - Windows 10/11 x64
  - Node.js v18+
  - ffmpeg (для RTSP-снимков iPanda):
       путь в .env как FFMPEG_PATH
  - Playwright Chromium (npm install скачает автоматически)
  - Доступ к камерам по сети (LAN / маршруты / VPN)
  - Inbound webhook в Bitrix24 (для загрузки снимков)


БЫСТРЫЙ СТАРТ
--------------
  1. npm install
  2. Заполни .env (см. CLAUDE.md → "Переменные окружения")
  3. .\setup-schedule.ps1      — создаст задачи Light + Daily
  4. .\menu.ps1                — для ручных проверок и настройки


ОТКАТ К v1
----------
Если v2 нужно откатить:
  1. Снять задачи v2: .\setup-schedule.ps1 -Remove
  2. Восстановить v1-папку из бекапа autocamera-BACKUP-*
  3. Восстановить v1 task scheduler из XML-бекапа
