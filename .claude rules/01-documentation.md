# 01. Документация — что и когда обновлять

Каждое изменение, которое **видит пользователь или оператор**, должно быть
отражено в документации **в том же изменении**, что и сам код. Не «потом»,
не «в следующем коммите». Если документация не обновлена — изменение
считается незавершённым.

## Карта документов

| Файл | Аудитория | Что туда |
|---|---|---|
| `CLAUDE.md` | будущие сессии Claude | поведение агента, архитектура, флаги CLI, env-переменные, версия проекта, секция «Что нового в vX» |
| `README.md` | разработчик / новый человек | стек, режимы запуска, меню `menu.ps1`, краткий список систем |
| `README.txt` | оператор без markdown | то же, что `README.md`, но текстом — синхронизировать |
| `autocamera-manual.docx` | заказчик / 1С | пользовательский мануал. Трогать только если меняется UX (email, helpdesk, меню) |
| `.env.example` | новый деплой | все новые env-переменные с placeholder-значениями и комментарием-блоком |
| `config/systems.json` | оператор | новая система / новый чекер / `extraCameras` |
| `config/schedule.json` | оператор | если меняются окна `light` / `daily` |

## Триггеры (когда обязательно править)

| Что изменил | Обнови |
|---|---|
| Добавил/удалил env-переменную | `.env.example` + соответствующий блок в `CLAUDE.md` («Окружение») |
| Добавил CLI-флаг (`--что-то`) | `CLAUDE.md` (раздел про флаги) + `README.md` (таблица «Режимы запуска») + при необходимости `menu.ps1` |
| Добавил новую систему камер / новый чекер | `CLAUDE.md` (раздел «Архитектура» + «Системы»), `README.md` (таблица «Системы камер»), `config/systems.json`, `.env.example` |
| Поменял поведение helpdesk / email | `CLAUDE.md`, `README.md`, `autocamera-manual.docx` |
| Поменял пункт меню `menu.ps1` | таблицу «Меню» в `README.md` и `CLAUDE.md` |
| Поменял версию | `CLAUDE.md` (шапка `**Версия:**`) + `README.md` (шапка) + добавь подраздел «Что нового в vX.Y» в `CLAUDE.md` и `README.md` |
| Изменил расписание / cron | `CLAUDE.md`, `README.md` («Режимы запуска») и `setup-schedule.ps1` |
| Удалил функциональность | везде, где она упоминалась — **поиском по слову** |

## Версионирование

- Проект ведёт версии вида `v2.1`, `v2.2`, ...
- Малая правка (фикс, рефактор без видимого эффекта) — версию не трогать.
- Новая видимая фича, новая система, новый CLI-флаг — поднять минорную
  версию (`v2.1 → v2.2`).
- Перелом совместимости (несовместимый формат `state/`, смена схемы
  `systems.json`) — мажорная (`v2 → v3`).
- При смене версии добавить раздел «Что нового в vX.Y» вверху `CLAUDE.md`
  и `README.md` (формат — см. как уже сделано для v2.1).

## Как обновлять `.docx`

`autocamera-manual.docx` — не трогать вручную через текстовый редактор.
Если нужно изменить — использовать `anthropic-skills:docx` (он умеет редактировать
Word корректно) или попросить пользователя. Никогда не перезаписывать целиком —
только точечные правки (find/replace, добавление параграфа).

## Чеклист перед сообщением «готово»

- [ ] Затронутые из таблицы выше документы обновлены.
- [ ] `CLAUDE.md` и `README.md` не противоречат друг другу.
- [ ] `.env.example` содержит все env-переменные, на которые ссылается код.
- [ ] Если поднимал версию — добавил блок «Что нового».
- [ ] Если правило поведения изменилось — обновил соответствующий файл
  в `.claude rules/` (см. [00-meta.md](00-meta.md)).
