# Правила работы Claude над проектом AutoCamera

Эта папка — обязательное чтение в начале каждой новой сессии.
Каждый файл = одно правило. Порядок чтения — по номерам.

| Файл | О чём |
|---|---|
| [00-meta.md](00-meta.md) | Как пользоваться правилами и как Claude сам их дополняет |
| [01-documentation.md](01-documentation.md) | Какие документы обновлять при изменении кода/поведения |
| [02-git.md](02-git.md) | Git-workflow: что коммитим, что никогда не заливаем, формат сообщений |
| [03-secrets.md](03-secrets.md) | `.env`, креды камер, IP внутренней сети — как не утечь |
| [04-style.md](04-style.md) | Русский язык, ES modules, поведение скрипта без человека |

Файл `Пример правил.txt` — образец формата из Cursor, оставлен для
справки. Сам по себе не применяется.

## Добавить новое правило

Если в разговоре всплыло что-то, что должно действовать в будущих
сессиях:

1. Найти подходящий файл по теме — дополнить.
2. Если темы нет — создать `NN-тема.md` со следующим свободным номером
   и добавить строку в эту таблицу.
3. Сказать пользователю одной строкой, что изменилось.

См. подробности в [00-meta.md](00-meta.md), раздел «Самодополнение
правил».
